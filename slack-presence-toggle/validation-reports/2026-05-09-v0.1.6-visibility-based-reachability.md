## Validation Report: slack-presence-toggle v0.1.6 (visibility-based reachability)

**Date**: 2026-05-09
**Status**: PASSED

### Scope

Two-part change tied together by a single user-visible narrative
("Slack is visually reachable" instead of "Slack has keyboard focus"):

1. **KWin script v0.1 → v0.2**: switches from `workspace.windowActivated`
   forwarding to a per-evaluation reachability check. Slack is reachable
   if at least one Slack window is on the current virtual desktop, not
   minimized, and not fully covered by any single higher-stacked window.
   Fixes the multi-monitor case where activating a window on the adjacent
   monitor takes keyboard focus but leaves Slack visually on top of its
   own monitor.
2. **State machine pre-grace-apply hook**: when the grace timer expires,
   run an optional hook (the orchestrator reloads the KWin script to
   re-emit the current reachability state) and defer the force-away by
   500 ms. If the FSM had drifted out of sync with reality and a fresh
   "reachable" event arrives in the deferred window, the deferred apply
   becomes a no-op. Defensive belt to the v0.2 KWin script's suspenders.

### Phase 4: Code Quality

- The deferred-apply path adds one new handle (`_deferred_apply_handle`)
  and one new method (`_do_grace_apply`). Cancellation is centralized in
  `_cancel_grace_timer` so transitions out of OTHER_FOCUSED_PENDING_AWAY
  abort both phases atomically — the comment in `_cancel_grace_timer`
  explains why both must be cancelled together (prevents a stale apply
  after focus has returned).
- The KWin script's reachability check explicitly does not handle
  multi-rectangle tiled coverage (two non-overlapping windows
  collectively covering Slack) — the comment in `isSlackReachable` calls
  this out and the README's troubleshooting section repeats the
  limitation. Decision is intentional (region-difference test would be
  too expensive on the hot path that fires on every geometry change).
- Per-window signal connects are wrapped in try/catch so a missing
  signal on any KWin build degrades to "we lose one trigger" rather
  than "the script refuses to load". `evaluate()` is idempotent — it
  early-exits when `reachable === lastReachable`, so over-firing is
  cheap.
- INFO-level transition logs added throughout the FSM. Over-logging is a
  conscious trade-off — focus-state drift is the single hardest class of
  bug to reproduce, so the journal trail is worth the noise.
