## Validation Report: slack-presence-toggle v0.1.2
**Date**: 2026-04-29
**Status**: PASSED

### Bug

After running cleanly for a day, the utility silently stopped working: tray
icon stayed green, no presence transitions fired, and no notifications.
Diagnosis showed the tray app was running and registered on the session
bus, the Slack token was still valid (`auth.test` ok), but
`qdbus6 org.kde.KWin /Scripting isScriptLoaded slack-focus-monitor`
returned `false` despite `~/.config/kwinrc` still having
`slack-focus-monitorEnabled=true`. KDE Plasma occasionally drops loaded
KWin scripts after Plasma reloads / logout cycles even when the kwinrc
Plugins entry is intact, and the app's previous behavior on detecting
this was to log a warning and give up.

### Fix

Three additions:

1. **Startup auto-heal**: `_reload_kwin_script_for_initial_focus` now
   loads the script from `~/.local/share/kwin/scripts/<name>/contents/code/main.js`
   when `isScriptLoaded` returns false (and the file exists on disk),
   rather than just logging a warning and returning. Same code path also
   handles "first run after install when KWin hasn't picked the script
   up yet."

2. **Periodic auto-heal**: a `QTimer` ticks every 5 minutes (when the
   FSM is enabled) and calls `_ensure_kwin_script_loaded`, which is a
   passive load-if-missing variant. Recovery emits an
   `urgency=low` notification ("Reconnected to KWin focus monitor.")
   so the user knows the system self-corrected.

3. **Manual recovery**: a new `Reload KWin script` menu item lives
   between `Disable auto-presence` and the Configure submenu. Clicking
   it calls the force-reload path (unload + load, which also re-emits
   the current focus event) and shows a confirmation notification.

The new path also handles the script-files-missing case: if the file at
`KWIN_SCRIPT_PATH` doesn't exist, the app raises an `urgency=critical`
notification telling the user to run `kwin-script/install.sh` and skips
the load attempt.
