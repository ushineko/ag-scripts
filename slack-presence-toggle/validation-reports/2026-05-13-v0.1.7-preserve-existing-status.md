## Validation Report: slack-presence-toggle v0.1.7 (preserve existing custom/scheduled status)

**Date**: 2026-05-13
**Status**: PASSED

### Scope

When the grace timer expires and the state machine applies the forced-away
state, it now reads the current Slack profile status via
`users.profile.get` first. If a non-empty status is present that is not
equal to the configured `status_text` (i.e., the user has a manual status
like "On vacation" or a Slack scheduled status that's currently active),
the status set is skipped. Only presence flips to `away`. The
`we_forced_status` flag stays false, so the subsequent clear path leaves
the user status untouched.

If `users.profile.get` itself fails (missing scope, transient error), the
code falls back to the prior unconditional set — this preserves the
feature for users without `users.profile:read` and matches the existing
best-effort semantics elsewhere in the state machine.

The matching acceptance criterion in
`specs/001-tray-focus-presence.md` ("If the user has set a non-empty
custom status before our utility activates, the utility leaves it alone
and only updates presence") was already marked complete in the spec but
was not actually wired up in code. This release closes that gap.

### Files changed

- `slack_presence_toggle/state_machine.py` — `_apply_forced_state` now
  calls `get_profile_status` and skips `set_profile_status` when a
  user-set status is detected
- `tests/test_state_machine.py` — 6 new cases:
  - user status present → status set skipped, presence still flips
  - Slack-scheduled status currently active → same skip path
  - existing status equals our text (leftover from prior session) →
    refresh as before
  - empty existing status → set ours as before
  - `get_profile_status` fails → fall back to set (don't break feature)
  - end-to-end skip-on-apply → no clear-side touches on focus return
- `tests/test_state_machine.py` — updated `test_focus_return_after_grace_clears_both`
  to expect `get_profile_status_calls == 2` (one on apply, one on clear)
- `slack_presence_toggle/version.py`, `pyproject.toml` — 0.1.6 → 0.1.7
- `specs/001-tray-focus-presence.md` — apply-forced-state section
  documents the new get-before-set step and the fallback behavior
- `README.md` — Usage section gains a paragraph explaining the
