**Date**: 2026-04-29

### Bug

v0.1.3 crashed at startup with:

```
TypeError: arguments did not match any overloaded call:
  connect(self, service, path, interface, name, slot): too many arguments
  connect(self, service, path, interface, name, signature, slot): argument 5 has unexpected type 'Application'
  connect(self, service, path, interface, name, argumentMatch, signature, slot): argument 5 has unexpected type 'Application'
```

The new `_wire_kwin_owner_watcher` in v0.1.3 used the Qt-style
`(receiver, slot_name_string)` overload for `QDBusConnection.connect`,
which the PyQt6 binding doesn't expose. The available overloads all
take a callable as the slot argument.

### Fix

One-line change: pass the bound method `self._on_dbus_name_owner_changed`
directly as the slot, instead of `(self, "_on_dbus_name_owner_changed")`.
PyQt6 marshals the signal arguments to the slot's parameters using the
`@pyqtSlot(str, str, str)` decorator that's already in place.
