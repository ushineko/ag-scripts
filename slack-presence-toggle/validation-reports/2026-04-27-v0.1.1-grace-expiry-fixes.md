## Validation Report: slack-presence-toggle v0.1.1
**Date**: 2026-04-27
**Status**: PASSED

Two bug fixes surfaced during initial v0.1.0 testing.

### Bug 1: No notification / tray refresh on grace-timer expiry

`FocusStateMachine._on_grace_expired` is invoked by the scheduler, which has
no return channel back to the orchestrator. As a result, the `TransitionResult`
from `_apply_forced_state` was logged but never reached `Application._after_transition`,
so:
- no desktop notification fired,
- the tray icon never recolored,
- the status header never refreshed,
- the API health state was never updated based on the call results.

**Fix**: added an optional `on_internal_transition` callback to `FocusStateMachine`,
invoked from `_on_grace_expired` after `_apply_forced_state`. `Application` wires
its `_on_internal_transition` method (which delegates to the existing
`_after_transition`) into the FSM at construction time. Method-driven transitions
(`on_window_activated`, `set_enabled`, `shutdown`) continue to return the result
directly to the caller, so the observer is reserved for genuinely internal
(timer-driven) transitions only.

### Bug 2: Startup with Slack unfocused never schedules grace timer

Initial state was `OTHER_FOCUSED` ("conservative"). Because `_handle_other_focus`
schedules a grace timer only when transitioning *from* `SLACK_FOCUSED`, a startup
with Slack already unfocused stayed at `OTHER_FOCUSED` forever and never forced
away — even after the KWin script's initial-active-window emit landed on the
listener.

**Fix**: changed initial state to `SLACK_FOCUSED`. On startup, the KWin script's
initial-active-window emit now lands as one of:
- Slack window: stays `SLACK_FOCUSED`, no transition needed
- Non-Slack window: transitions to `OTHER_FOCUSED_PENDING_AWAY` and schedules
  the grace timer, which fires and applies forced state correctly

Removed the `__SCRIPT_LOADED__` sentinel send from the KWin script body (legacy
prototype debugging; now redundant and would otherwise cause a spurious timer
schedule that's immediately cancelled by the real focus event milliseconds later).
