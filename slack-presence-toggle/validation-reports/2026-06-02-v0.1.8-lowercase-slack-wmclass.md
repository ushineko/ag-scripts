## Validation Report: slack-presence-toggle v0.1.8 (lowercase Slack WM_CLASS fix)

**Date**: 2026-06-02
**Status**: PASSED

### Scope

The official Slack desktop client (4.49+) changed its window `WM_CLASS` /
`resourceClass` from `Slack` to lowercase `slack`. The KWin script matched
the resource class case-sensitively against the constant `SLACK_CLASS =
"Slack"`, so no window matched, `isSlackReachable()` always returned
`false`, and the tray app treated Slack as permanently unreachable (never
flipping presence back to active).

The fix makes window matching case-insensitive in two places:

- KWin script: compares `resourceClass.toLowerCase()` against a normalized
  constant. The constant is still the sentinel emitted to the tray app on
  the reachable path, so it stays paired with `slack_resource_class`.
- State machine: compares the incoming `resource_class` against
  `slack_resource_class` case-insensitively, as a second layer so a
  config/sentinel case mismatch cannot silently break detection again.

Root cause confirmed on this machine: a live KWin window dump reported
`rc='slack'` for the Slack client (version 4.49.89). After the fix, a
diagnostic mirroring the reachability match found the `slack` window (not
minimized, on current desktop, valid geometry).

### Files changed

- `kwin-script/contents/code/main.js` — case-insensitive `resourceClass`
  match via a normalized `SLACK_CLASS_LC` constant; comment updated to
  document the `Slack`/`slack` tolerance
- `slack_presence_toggle/state_machine.py` — `on_window_activated`
  compares classes with `.lower()` on both sides
- `slack_presence_toggle/version.py` — 0.1.7 → 0.1.8
- `README.md` — Troubleshooting note + new Changelog section (and TOC
  entry) documenting the class-case change

- **Additive**: no APIs removed, no config fields changed, no schema
