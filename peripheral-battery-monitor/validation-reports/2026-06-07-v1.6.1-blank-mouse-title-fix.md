# Validation Report — v1.6.1 Blank Mouse Title Fix

**Date**: 2026-06-07
**Version**: 1.6.1
**Scope**: Fix blank Logitech mouse title that appeared on desktop startup after reboot and persisted until the monitor was restarted.

## Summary

At login the Logitech receiver (and solaar's HID stack) is often not fully enumerated when the first battery poll runs. Solaar lazily resolves and caches `dev.name` on the device object; a poll during that window can latch a blank name that never recovers because the cached object is reused every poll. The fix evicts the cached device when its name resolves blank so the next poll rebuilds a fresh object and re-resolves the name, normalizes names at the source, and adds a UI fallback so a blank name can never render an empty title.

## Changes

- `battery_reader.py`
  - `_extract_battery`: `device_name=(dev.name or "").strip()` — normalize name at the source.
  - New `_name_is_blank()` helper.
  - `get_mouse_battery` cached path: evict `_CACHED_MOUSE` (and close the receiver) when a cached device returns battery data with a blank name, forcing a fresh rebuild next poll. Offline reads (`None`) are left untouched, preserving offline-cache behavior.
  - `get_mouse_battery` scan path: do not retain a freshly-built device whose name resolves blank.
- `peripheral-battery.py`
  - `_update_label_block`: `disp_name = (info.device_name or "").strip() or fallback_name` — blank/whitespace name renders the fallback label.
  - Version bump to 1.6.1.
- Tests: 5 added in `tests/test_battery_logic.py` (name strip, blank→empty normalization, cache eviction on blank name, cache retention on good name, UI fallback on blank name).
