# Validation Report — v1.8.0 Window Position Save/Restore (KDE Wayland)

**Date**: 2026-07-17
**Version**: 1.8.0
**Scope**: Make the window reappear at its last on-screen position on the next fresh launch, on KDE Plasma 6.7 Wayland.

## Summary

The app opened at a default position every launch. The previously installed KWin
rule set `positionrule=4` ("Remember"), but that never worked: KWin position
rules on Wayland only select the screen and snap to its origin, and the app also
called `self.move(100, 100)`, which the compositor ignores.

Empirical testing on the live session (KWin 6.7.2 / Qt 6.10) established the real
constraints and the working mechanism:

- `QWidget.move()` is ignored by KWin — clients cannot self-position.
- `windowHandle().geometry()` returns a bogus value: Qt reported `(2560,0)` while
  KWin's true `frameGeometry` was `(3180,1240)`. The compositor is the only source
  of truth.
- `QMoveEvent` does not fire for compositor-driven moves (interactive
  `startSystemMove()` drags included), so it cannot trigger a save.
- KWin `position` rules (Force/Remember/Apply) snap to the screen origin, not the
  exact coordinate.
- A loaded KWin script runs only once, so each report is a fresh load/run/unload.
- Native session restore (`xx-session-management-v1`) needs Qt 6.12+ (Arch ships
  6.11.1; the app runs pip PyQt6 → Qt 6.10) and is opt-in/experimental — not usable.

The working approach, now the repo-wide reference for custom apps: drive the KWin
Scripting D-Bus API in-process. Restore sets `frameGeometry` from saved coords; a
report script reads the true geometry and calls back over D-Bus, which is
persisted. Save is triggered by a short poll burst after a drag (no idle cost).

## Changes

- `kwin_window_position.py` (new) — reusable `KWinWindowPosition` helper:
  `restore(x, y)`, `request_report()` → `geometryReported(x, y)` signal,
  `is_available()`. In-process QtDBus; degrades to no-op off KDE.
- `peripheral-battery.py`
  - Adds `window_x`/`window_y` to settings (default `None`).
  - Constructs `KWinWindowPosition`, restores position ~350 ms after map, and
    persists the true position via a settle-poll burst keyed off `mousePressEvent`.
  - Removed reliance on `moveEvent`/`move()` for positioning. Version → 1.8.0.
- `install_kwin_rule.py` — stops setting the dead `positionrule`/`sizerule`/
  `screenrule` remember rules and clears any a prior version left behind.
- `README.md` — version, Features (Position Restore), Changelog v1.8.0.
- `.claude/CLAUDE.md` — new "Window Position Save/Restore" subsection documenting
  the repo-wide approach.
- Tests — new `tests/test_kwin_window_position.py` (9); `tests/test_battery_logic.py`
  updated to stub the new module and restore shadowed Qt modules in
  `tearDownModule` (fixes a latent test-isolation crash).

## Phase 3 — Tests

- `QT_QPA_PLATFORM=offscreen /usr/bin/python3 -m pytest tests/` — **76 passed**
  (67 pre-existing + 9 new). Run twice, deterministic; `test_kwin_window_position.py`
  alone: 9 passed.
- Live verification on the running session (KWin 6.7.2 Wayland):
  - **Restore**: seeded `window_x/y = 640,1550`, launched the real app, queried
    KWin — window at `640,1550`.
  - **Save**: after an external move to `(1120,1660)`, the settle-poll → report →
    persist wrote `window_x/y = 1120,1660` to the config (faithful harness mirroring
    the app's wiring).
- Note: unit tests cover pure logic and graceful degradation; the KWin round-trip
  is an integration boundary requiring a live compositor and is covered by the live
  checks above (not runnable in CI without KWin).

## Phase 4 — Code Quality

- New helper is self-contained, single-responsibility, and written to be copied
  into other apps. No dead code; the obsolete `moveEvent` handler was removed, not
  left dangling. JS generation isolated in `_move_js`/`_report_js`.

## Phase 5 — Security

- No new external dependencies (`PyQt6.QtDBus` ships with the already-used PyQt6);
  dependency scan not applicable. No secrets/credentials.
- No shell execution — KWin is driven over D-Bus, not a subprocess. Temp JS files
  use `tempfile.mkstemp` (0600) and are unlinked in `finally`.
- Injection surface: the KWin JS embeds `app_id` (a hardcoded trusted constant) and
  integer coordinates (`int()`-cast). No user-controlled string reaches the JS.
- The `ReportGeometry(int, int)` D-Bus method is callable by any session-bus peer;
  worst case a hostile peer sets a bogus saved window position (session bus is the
  per-user trust domain). Low severity, no data exposure.

## Phase 5.5 — Release Safety

- **Rollback**: revert the commit. Self-contained; adds two optional config keys
  (`window_x`/`window_y`) that older versions ignore. No migration, no schema break.
- **Additive**: the only removals are dead KWin remember rules and the no-op
  `move()`/`moveEvent` positioning, both superseded in the same release. Off KDE the
  feature is a no-op; existing behavior is preserved.
