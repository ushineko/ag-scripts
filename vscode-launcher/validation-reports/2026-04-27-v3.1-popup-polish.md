## Validation Report: Popup Polish (v3.1)
**Date**: 2026-04-27
**Spec**: 012-popup-mode (post-implementation iterations)
**Status**: PASSED

### Summary

Iterative polish on the v3.0 popup based on live testing feedback. No new
features; four behavioral / visual fixes plus one internal refactor.

### Phase 3: Tests
- Test suite: `/usr/bin/python3 -m pytest tests/`
- Results: 154 passing, 0 failing
- Test changes:
  - `tests/test_unit_popup.py`: removed the prior `mouse_entered` /
    `mouse_left` tests; added two `mouse_moved` tests (one for the
    popup itself via `mouseMoveEvent`, one for the QListWidget viewport
    via `eventFilter`). The viewport route was added after the user
    reported that "keep moving" wasn't restarting the timer in
    practice — `QListWidget` delivers mouse moves through its
    `viewport()` child, not through the list widget itself, so the
    popup now installs the filter on both.
- Live verification:
  - Popup pops up under stationary cursor → counts down normally (does
    not get pinned open).
  - Mouse motion over the popup or a row → timer restarts; click
    commits.
  - Hint text now reads "Tap hotkey to cycle    Pause or click to
    commit" and aligns with list rows.
  - Frame border no longer cascades onto descendant labels.
  - Alt-Tab MRU verified by hand: A→D→A flip-flop works; selecting C
    after that builds [C, A, D] and the next popup shows previous (A)
    at row 0, current (C) at row 1.
- Status: PASSED

### Phase 4: Code Quality
- Removed:
  - `_activated_at_by_path: dict[str, float]` instance attribute and
    all three writes (in `_on_activate_requested`, `_launch_or_activate`,
    `_launch_paths`)
  - `mouse_entered` / `mouse_left` signals on `WorkspacePopup`
  - The `enterEvent` / `leaveEvent` handlers on the popup
  - Two unused popup imports (`QApplication`, `QHBoxLayout`) — caught
    in the v3.0 quality pass; verified still gone.
- Added:
  - `_mru_stack: list[str]` instance attribute
  - `_push_mru(path)` helper (move-to-top, no dupes)
  - `mouse_moved` signal on `WorkspacePopup`
  - `mouseMoveEvent` override and `eventFilter` on the popup
- Reworked `_sort_alt_tab_order` to read from the MRU stack rather than
  reconstructing a "current" workspace from timestamps. The new sort
  is pure: `O(N)` to project the live MRU + `O(N log N)` for the final
  stable sort, no max-by-key reduction.
- Stylesheet scoped to `QFrame#popupFrame` so `QLabel`s (which inherit
  `QFrame`) no longer pick up the 1-pixel border. Title/hint label
  padding aligned to 10 px to match list-row padding.
- Duplication: none.
- Encapsulation: the popup's mouse-tracking concern is fully contained
  in `WorkspacePopup`; `MainWindow` consumes a single signal.
- Status: PASSED

### Phase 5: Security Review
- No new external surfaces: the changes are entirely in-process (Qt
  signal/slot wiring, list mutation, stylesheet scoping).
- Removed surface: none.
- No hardcoded secrets, no new dependencies, no user-input sinks.
- Status: PASSED

### Phase 5.5: Release Safety
- Change type: UI-only iteration on top of v3.0.
- Rollback plan: `git revert HEAD` restores v3.0. Config schema
  unchanged (the `global_hotkey` and `popup_commit_delay_ms` fields
  added in v3.0 are unchanged in v3.1; nothing new persisted to disk).
- Behavior changes visible to users:
  - The popup's row order now reflects an MRU stack rather than
    VSCode's recents within the running group. Users who had no
    activation history yet (cold daemon) see the same order as
    before; once they activate workspaces via the popup, the stack
    builds up and the running-group order reflects their switching
    history.
  - The popup is no longer pinned open just by having the cursor over
    it. Users who relied on hover-to-pin will need to nudge the mouse.
- Risks: low. Worst case is a UX preference issue, mitigated by the
  config-driven `popup_commit_delay_ms` (users can dial up the delay
  if "tap once and forget" feels too eager).
- Status: PASSED

### Phase 6.5: Spec Reconciliation
- Spec 012 was already marked COMPLETE in v3.0 with a "Post-implementation
  deltas" section. This iteration adds three more deltas to that list
  (Alt-Tab MRU stack replacing activation timestamps, mouse-motion-as-
  activity, frame style scoping). No re-opening of spec 012 needed —
  these are layered refinements that preserve the spec's high-level
  intent. Acceptance criteria remain satisfied; in particular:
  - "Activated workspaces ... bubble to the top of the running group"
    is now realized via the MRU stack rather than a per-path
    timestamp dict, which gives the desired Alt-Tab two-window flip
    semantics that timestamps alone could not.
- Status: PASSED
