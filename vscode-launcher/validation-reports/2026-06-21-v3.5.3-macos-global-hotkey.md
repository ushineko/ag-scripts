## Validation Report: v3.5.3 macOS global popup hotkey
**Date**: 2026-06-21
**Commit**: pre-commit
**Status**: PASSED (with a manual-test handoff — see Limitations)

### Problem
The global popup hotkey was KGlobalAccel-only (KDE D-Bus), so it did nothing
on macOS.

### Implementation
New `macos_global_shortcut.py`: a Carbon `RegisterEventHotKey` backend via
`ctypes`, exposing the same `pressed` / `released` signals and
`is_available()` / `set_binding()` / `unregister()` surface as the KDE
`GlobalShortcut`, so the launcher selects a backend by platform with no other
changes. Carbon was chosen because `RegisterEventHotKey` needs no Accessibility
permission (unlike a CGEventTap) and emits both press and release events (the
tap-to-cycle popup needs the pair). The handler runs on the main CFRunLoop —
the loop Qt drives on macOS — so emitting Qt signals from it is thread-safe.
All ctypes symbols have explicit argtypes/restypes (64-bit pointer safety) and
every failure path degrades to `is_available()`/`set_binding()` returning False.

The macOS default hotkey is `⌘⇧Space` (`Meta+Shift+Space`); the Linux default
`Shift+Tab` is reserved by macOS. A config carried over from Linux (hotkey ==
`Shift+Tab`) is migrated to the macOS default on load.
