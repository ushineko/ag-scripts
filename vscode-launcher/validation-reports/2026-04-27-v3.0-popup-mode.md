### Phase 5: Security Review
- New protocol surfaces:
  - **KGlobalAccel** (already validated in spike 1, now productionized).
    The D-Bus calls are scoped to a fixed component name owned by this
    user; no untrusted-input paths. Hotkey strings come from
    `workspaces.json` (user-edited config) and from the `QKeySequenceEdit`
    widget; both are passed through `QKeySequence.fromString().toCombined()`
    which produces a well-typed integer or `None` (rejected). Type
    marshaling uses explicit `QDBusArgument` + `QMetaType.Type.*.value`
    so PyQt6 cannot promote untyped data into the D-Bus message
    accidentally.
  - **session-bus singleton** (`org.kde.vscode_launcher`). The exposed
    `ShowMainWindow` method takes no arguments and only triggers a
    UI-thread call into `MainWindow._show_main_window`. No
    user-controlled input is processed.
- Removed surface: none.
- Dependency scan: no new third-party dependencies. PyQt6 / PyQt6.QtDBus
  are already required for the launcher GUI.
- No hardcoded secrets. No credentials anywhere in the new code.
- Status: PASSED

### Phase 5.5: Release Safety
- Change type: feature addition (tray daemon, global hotkey, popup) +
  a behavioral default change (tray-resident is now the only mode).
- Rollback plan: `git revert HEAD` restores v2.0. No persistent state
  changes that would break a downgrade — config additions
  (`global_hotkey`, `popup_commit_delay_ms`) are ignored by v2.0's
  `ConfigManager`, which preserves unknown keys on save. Autostart
  entry written by `install.sh` is removable via `uninstall.sh` or
  manually.
- Behavior changes visible to users:
  - `vscode-launcher` no longer exits when the main window is closed —
    it hides to the tray. Explicit Quit from the tray menu exits.
  - A `Shift+Tab` global hotkey is registered system-wide while the
    daemon is running; if any application relied on `Shift+Tab` for
    backtab navigation, that interaction now goes to the launcher.
    Mitigation: rebind via Settings.
  - Autostart entry installs at login (one-time prompt could surprise
    users on next reboot); install.sh prints a hint about this.
- Risks mitigated:
  - If KGlobalAccel registration fails (combo conflict, service down),
    the daemon prints a warning and continues running — the main window
    and per-row actions still work; only the popup is degraded. User
    can rebind via Settings.
  - If the session bus is unreachable, `SingletonGuard.claim()` returns
    True (no IPC possible). Single-instance enforcement is best-effort.
  - On Wayland, `Qt.WindowType.Popup` would error with "Failed to
    create grabbing popup" because tray daemons have no `transientParent`
    with input. The popup uses `Qt.WindowType.Tool` instead, with
    inline rationale.
- Status: PASSED
