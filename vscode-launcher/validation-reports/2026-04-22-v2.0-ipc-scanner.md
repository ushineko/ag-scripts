### Phase 5: Security Review
- New protocol surface: AF_UNIX socket to VSCode's main process
  - Target socket is under `$XDG_RUNTIME_DIR` (per-user runtime dir,
    not globally writable) — no risk of foreign-user spoofing
  - Socket contents are VSCode's own data; parsed with VSBuffer
    deserialization, never with eval / exec
  - JSON payloads are parsed with `json.loads` (safe by default)
  - All decode / parse errors wrapped in try/except → return None;
    no exception escape path that reaches the user
- Removed surface: KWin scripting via qdbus6 (still used for actions)
  and journalctl reads (entirely gone from the scan path)
- Temp file creation: gone from the scan path (was only needed by the
  old KWin-script-load approach); actions still use `tempfile.NamedTemporaryFile`
  with unique prefix as before
- No new user-input sinks. Labels / paths still flow through subprocess
  list-form argv; never shell-interpreted.
- Socket permission check happens implicitly. The launcher process has to be
  the same user VSCode runs as for the XDG_RUNTIME_DIR to be readable
- Status: PASSED

### Phase 5.5: Release Safety
- Change type: major backend swap (scanner implementation)
- Rollback plan: `git revert HEAD` restores v1.8.1. No on-disk state
  changes. Config schema unchanged. Desktop entry unchanged. Install
  script's zsh hook unchanged.
- Behavior changes visible to users:
  - Scanning is effectively instant now (imperceptible vs v1.8.1's
    occasional ~500 ms stalls)
  - Per-window launch times are now different per running window (was
    same for all in v1.8)
  - Under heavy system load, v1.8.1 could still occasionally miss a
    state flip due to journalctl flush race — v2.0 can't, the IPC
    response is atomic
- Risks mitigated:
  - If VSCode changes its internal IPC protocol: scan returns None,
    UI shows stale state until user clicks Refresh (same degradation
    as v1.8.1 would exhibit on KWin script API changes)
  - If the socket is permission-denied (unlikely — it's per-user):
    scan returns None, same degradation
- Status: PASSED
