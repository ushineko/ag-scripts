### Problem
After installing the v3.5 `.app`, launching it from Spotlight did nothing
visible and the menu-bar icon was not findable (obscured by the display
notch). Root cause: on macOS there is no D-Bus session bus, so the Linux
single-instance "signal the running daemon to show its window" path can't
fire; and LaunchServices delivers a relaunch of the already-running menu-bar
agent as an *activation* of the existing process, not a new launch. Nothing
handled that activation, so the hidden window never surfaced.

### Fix
`main()` (macOS only) connects `QApplication.applicationStateChanged` and
shows the main window when the app becomes active, after a 1.5s settle delay
(so login `--tray` autostart stays hidden) and only when the window is
currently hidden (so it doesn't fight a menu-bar interaction with an
already-open window).
