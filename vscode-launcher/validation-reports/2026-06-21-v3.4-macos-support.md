## Validation Report: v3.4 macOS support (spec 013)
**Date**: 2026-06-21
**Commit**: pre-commit
**Status**: PASSED

### Scope
Completes spec 013 — macOS support for vscode-launcher. Continued from a prior
session that had implemented only the macOS IPC-socket discovery branch in
`platform_support.py`. This session added: `process_start_time` via `psutil`,
a macOS menu-bar template icon, `install.sh`/`uninstall.sh` macOS branches with
a LaunchAgent, platform-resolution tests, and README/version updates.

### Phase 3: Tests
- Command: `QT_QPA_PLATFORM=offscreen python3 -m pytest -q`
- Results: **168 passing, 0 failing**
- New file: `tests/test_unit_platform_support.py` (state DB / IPC socket /
  process-start-time / config-dir resolution across Linux, macOS, Windows).
- Fixed 7 pre-existing `test_unit_vscode_ipc.py` tests that assumed Linux
  socket discovery and failed on macOS: 3 from the platform branch scanning
  `~/Library` instead of `$XDG_RUNTIME_DIR`, and 4 from macOS's ~104-byte
  AF_UNIX path limit (pytest's `tmp_path` is far longer). A new
  `linux_runtime_dir` fixture pins the Linux branch and supplies a short-path
  runtime dir, so the suite is green on both platforms.
- Status: PASSED
