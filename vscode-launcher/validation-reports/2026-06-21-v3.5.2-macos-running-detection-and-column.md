**Date**: 2026-06-21
**Commit**: pre-commit

### Problems (reported from the running macOS app)
1. No VSCode windows detected as running — every workspace showed `Start`,
   none showed `● running`, despite two VSCode windows being open.
2. The Workspace name/path column was visually cut off.

### Root causes & fixes
1. `caption_matches_label` token-split window titles on `" - "` (hyphen) only.
   macOS VSCode renders the title separator as an em-dash (`" — "`), so the
   project token never split out and never matched. Confirmed live:
   `list_vscode_windows()` returns the open windows with em-dash titles. Fix:
   split on hyphen / en-dash / em-dash via a regex (`_CAPTION_SEPARATORS`).
2. Default window geometry was 700×500, but the five fixed columns total 570px
   (check 40 + status 90 + launched 100 + tmux 160 + actions 180), collapsing
   the stretchy Workspace column to ~130px. Fix: default 1180×600 and a 900px
   minimum width.
