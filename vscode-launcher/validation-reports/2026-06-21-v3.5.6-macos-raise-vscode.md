**Date**: 2026-06-21
**Commit**: pre-commit
**Status**: PASSED

### Problem
After v3.5.5, activating a running workspace still didn't visibly raise VSCode.
`code <path>` focuses the right window *inside* VSCode, but the launcher runs as
a background `LSUIElement` agent and macOS blocks a background app from pulling
another app to the foreground — so VSCode focused the window but stayed behind.

### Fix
`Launcher.focus_workspace` now, after `code <path>`, also runs
`open <Visual Studio Code.app>`. `open` activates via LaunchServices, which
works from any process context (including a background agent). The `.app` path
is resolved from the `code` CLI symlink (`_vscode_app_path`), so it follows
whatever editor `code` points to (VSCode / Insiders / VSCodium).
