### Phase 5: Security Review
- No new subprocess surface; no new file I/O; no new env-var handling
- The UI refactor is purely PyQt widget swapping — attack surface unchanged from v1.4
- `setCellWidget` receives only the launcher's own `QLabel` / `QCheckBox` / `QWidget` subclasses; no user input flows into the widget construction
- Rich-text labels use hardcoded HTML templates with workspace data interpolated — labels come from VSCode's state.vscdb and are already trusted to be filesystem paths (not arbitrary attacker input). Qt's QLabel rich-text renderer doesn't execute JS.
- No regression from v1.4's security posture
- Status: PASSED
