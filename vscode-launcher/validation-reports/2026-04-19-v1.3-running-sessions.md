### Phase 5: Security Review
- New surfaces:
  - Temp JS script written to `tempfile.NamedTemporaryFile` with prefix `vscode-launcher-scanner-` — standard mkstemp-equivalent semantics, no TOCTOU risk (exclusive create, unique name)
  - `qdbus6 org.kde.KWin ...` calls use list-form argv with hardcoded paths — no shell interpretation, no injection path
  - `journalctl` invocation filtered to the user's own `plasma-kwin_wayland` unit, read-only
  - JSON parsing from journal via `json.loads` (no pickle, no eval); malformed payload safely returns None
- All subprocess calls use `check=False` and `timeout=5` — no hang, no exception leak
- Script unload runs in `finally` so we don't leak named scripts in KWin's registry on error
- Script id from qdbus6 stdout is coerced via `int()` and range-checked (`< 0`) before use in the D-Bus path `/Scripting/Script<id>`
- No new secrets, no credentials paths
- Status: PASSED
