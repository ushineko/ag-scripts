### Bisect Findings (locked the wine vs Proton vs Steam Input vs HDR matrix)

| Combo | HDR | Controller |
| --- | --- | --- |
| System wine + wrapper | ❌ | ✅ always |
| Proton-GE + wrapper, Steam off | ✅ | ✅ |
| Proton-GE + wrapper, Steam on | ✅ | ❌ — buttons no-op |

Root causes:
- HDR through gamescope's WSI layer requires Proton-GE (system wine 11.x doesn't propagate the HDR signal cleanly through DXVK).
- Proton-GE's `winebus.sys` enumerates joysticks via udev and can't disambiguate when Steam Input's virtual `Microsoft X-Box 360 pad` (vendor `0x28de`, product `0x11ff`) is present alongside the real 8BitDo. v3.0.1's `SDL_JOYSTICK_BLACKLIST_DEVICES` env var didn't help because winebus doesn't go through SDL2.
- System wine's gamepad path handles the duplicate gracefully.

User chose controller over HDR. v3.1.0 implements that choice.
