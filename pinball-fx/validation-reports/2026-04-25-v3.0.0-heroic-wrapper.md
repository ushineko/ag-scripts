### Phase 5: Security Review

**Tooling**:
- `pip-audit --no-deps` against `PyQt6`: no known vulnerabilities.
- `pyflakes` against all Python sources: clean.
- `shellcheck` against `pinball_fx_wrapper.sh`, `install.sh`, `uninstall.sh`: clean.

**Secrets check**: `grep -rEn "(password|secret|token|api[-_]?key|BEGIN [A-Z]+ PRIVATE KEY)"` — no hardcoded credentials. Only matches were prose in README explaining the Epic Online Services auth issue.

**OWASP review (changed code only)**:
- **A01 Broken Access Control**: All file writes restricted to `~/.config/kwinrulesrc` (default) or caller-supplied path (tests only).
- **A03 Injection**: Bash wrapper passes Heroic's args via `"$@"` (correct quoting). gamescope/screen-detect calls use list form (no `eval`, no `bash -c`). Python helper uses no shell. KWin installer uses Python's `subprocess.run` with list args.
- **A05 Security Misconfiguration**: Adds a single KWin rule. No new ports, services, or auth surfaces.
- **A06 Vulnerable Components**: PyQt6 only third-party dep; pip-audit clean.
- **A08 Integrity**: No code download/exec from network.
- **A09 Logging Failures**: Wrapper logs the resolved gamescope cmd line + the args Heroic passed (which include Epic auth tokens like `-AUTH_PASSWORD=…`). The logs go to stdout/stderr that Heroic redirects to its per-game log; same logs Heroic itself writes when launching without our wrapper. No new disclosure.
- A02, A04, A07, A10: not applicable to a desktop launch wrapper.

**Limitations**: This is an AI-assisted review of changed code, not a compliance artifact.

- Status: PASSED
