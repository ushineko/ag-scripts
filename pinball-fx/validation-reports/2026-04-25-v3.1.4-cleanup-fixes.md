### Scope

Four incremental wrapper fixes layered on top of v3.1.0's "drop HDR" pivot. None changed the architecture; each fixed a bug surfaced by real-world testing.

| Version | Fix |
| --- | --- |
| v3.1.1 | `-W -H` vs `-w -h` were inverted (`PINBALL_FX_WIDTH` set the output buffer instead of the nested resolution the game sees). Added EXIT trap to sweep orphan `gamescopereaper`. |
| v3.1.2 | v3.1.1's EXIT trap never fired in practice. `wait $GS_PID` blocked indefinitely because `gamescopereaper` is a subreaper waiting for wine daemons (`winedevice.exe` etc.) to exit. Added a background watchdog that takes gamescope down once the actual game `.exe` exits. |
| v3.1.3 | Heroic still showed "playing" because it polls for any process owning the game's `WINEPREFIX`. Cleanup now runs `wineserver -k` for the inherited `WINEPREFIX`. |
| v3.1.4 | v3.1.3 wasn't enough: (a) reaper pkill pattern matched `PinballFX-Win64-Shipping.exe` but the reaper's cmdline references the launcher `PinballFX.exe` â€” never matched. (b) `wineserver -k` is a no-op when no wineserver is running (which is exactly the post-game-exit state). Cleanup now uses the launcher pattern for the reaper kill, and falls back to SIGKILL'ing wine system daemons (`winedevice|services.exe|plugplay.exe|explorer.exe|wineserver`) whose `/proc/PID/environ` shows matching `WINEPREFIX`. |
